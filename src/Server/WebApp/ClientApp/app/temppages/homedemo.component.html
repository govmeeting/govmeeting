Govmeeting - under construction

This is an open source project whose goal is to involve people more in their local governments.
The Github project is at: Govmeeting Github Project

Pages with an asterisk * show parts partially completed. Other pages describe parts to be built.

About

The About page explains more about the goals of project.

Fix ASR *

Govmeeting will retrieve meeting recordings, extract the audio and use the Google Auto Speech Recognition to create a transcript. However, not all the processing is automatic. A couple of steps need the help of community volunteers. The first step is fixing errors in the generated text.

The Fix ASR page demos a program for making corrections. A user watches a section of the meeting and corrects the corresponding text. There are two edit modes to facilitate this:
  • Insert Mode: A cursor appears when clicking in the text. What you type is inserted.
  • Replace Mode: Click to highlight a word. What you type replaces the word. A space deletes it. The next word is then highlighted. This mode makes it easier to replace of delete incorrect words.
The software assigns small portions of a meeting to multiple volunteers.

Add Tags *

The second manual step that must be done is to assign "tags" to parts of the meeting:
  • Section tags: For example: Reading of Minutes, Reading of Bills, Public Comments, etc. Pre-processing attempts to insert these but sometimes it is not entirely accurate.
  • Topic Tags: These indicate the start of a new topic of discussion. For example: "Repaving of Main St.", "Hiring of two policemen", etc.
The Add Tags page shows a program that tries to provide a quick and easy way to do this:
  • Highlight a phrase of the text to use it as the topic name.
  • Select a topic name from a drop-down combo box. These are the topics from other recent meetings.
  • The user can click the up or down arrows to move section or topic names.
It is important to assign clear topic names. The topic names are be used for:
  • Searching the text of prior meetings for discussion on ths this topic.
  • Setting alerts for when this topic is discussed at future meetings.

Existing Transcripts

Some larger cities provide transcripts. But they are often not user friendly. They first need to be processed and put in the same format as those created from recordings. Then, like the others, we need to add tags. The Existing Transcripts page shows an example of a transcript provided by larger city.

View Meeting *

After a transcript is processed, a browsable transcript is made available. There are many ways to display it. The View Meeting page is a quick and easy demo that was built that allows you to:
  • Click on a topic and see only what was said about that topic during the meeting.
  • Click on a speaker name and see only what that person said during the meeting.
To be more useful, it should display some of the surrounding context to what they said. It should also allow you to choose the section of the meeting that you want to view (Reading of Minutes, Public Discourse, etc.)

Search Meetings

After the meeting information is completely processed, the data is extracted and used to populate a relational database. Then it becomes easy to use the data in many ways.
  • You can search by issues, topics, speakers, bills, words said or any other category that we choose to save.
  • You can have the system alert us when specific events occur.
  • You can compare how similar issues are handled by other communities around the world.
