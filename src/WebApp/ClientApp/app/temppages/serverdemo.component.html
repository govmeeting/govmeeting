Server Processing

Step 1 is to retrieve and process existing recording and transcripts.

Process Video recordings

The following processing occurs for video recording of meetings. [An asterisk * means code is written for this.]

  • Retrieve new recordings as they become available.
  • Convert to the primary web video formats (mp4, ogg and webm).
  • Extract and merge the audio tracks. *
  • Uploaded the audio file to the Google cloud. *
  • Call the Google Speech API to do auto voice recognition. *
  • Split the video, audio and trancript files into 3 minute work segments. *
  • Serve the work segments to the second step - Correcting errors. *

Post processing

Whether the original information came from a recording or an exisiting transcript, the following post processing occurs.

  • Transform the data into an object representation that is used to populate a database.
  • Provide a variety of ways to search, access and display the information.
  • Accept personal search requests that it will store for a user and run against future data that it adds to the database.
  • Alert users when their personal search requests yield results on new data.
